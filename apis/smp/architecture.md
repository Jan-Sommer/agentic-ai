## PCI Stack

| Component | Tech | Port | Role |
|-----------|------|------|------|
| `api-gateway` | NestJS + Apollo Gateway | 3000 | GraphQL federation entry point + REST proxy |
| `data-management-api` | NestJS + Apollo Federation | 3001 | GraphQL subgraph + REST SSE/export |
| `carbon-footprint-service` | NestJS + BullMQ | 3002 | Background emission calculation worker |
| `ui-pci` | Angular | 4200 | Frontend |

**Databases:** MongoDB (primary), PostgreSQL (annual metrics)  
**Queue:** Redis + BullMQ  
**Auth:** Keycloak JWT

## Waves PHP Backends (common pattern)

All five PHP backends share the same skeleton:

| Aspect | Value |
|--------|-------|
| Language | PHP 8.4+ |
| Framework | Symfony 7.4 |
| API layer | API Platform 3.4.17 |
| Database | PostgreSQL + Doctrine ORM |
| Auth | JWT via `lexik/jwt-authentication-bundle` |
| File storage | Azure Blob Storage (`league/flysystem`) |
| Async | Symfony Messenger |
| Exports | mPDF (PDF), PhpSpreadsheet (Excel), CSV |
| Health | `GET /api/health` — public |
| OpenAPI | Auto-generated by API Platform at `/api/docs.json` |

## Lambda / Azure Function Stack

| Repo | Platform | Runtime | Trigger |
|------|----------|---------|---------|
| `waves-cft` | AWS Lambda | Python 3.11 | HTTP (API Gateway + API key) |
| `waves-tendering-api` | AWS Lambda | Python 3.12 | HTTP (API Gateway + API key) |
| `waves-cft-mapping-sam-azure` | Azure Functions | Python 3.9 | HTTP (Managed Identity + API key) |
| `waves-cft-parse-excel-azure` | Azure Functions | Python 3.9 | Blob trigger (client-ingress container) |
| `waves-cft-tendering` | AWS Lambda | Python 3.8 | Compute only (no HTTP trigger) |

## Authentication

| Product | Mechanism | Details |
|---------|-----------|---------|
| PCI | JWT (Keycloak OIDC) | `Authorization: Bearer {token}` — forwarded through Apollo Gateway |
| All Waves PHP | JWT (lexik/jwt-auth) | `Authorization: Bearer {token}` — shared public key `JWT_PUBLIC_KEY` |
| waves-smp-backend | JWT + API Key | `Authorization: Bearer` or API key in header |
| waves-tendering-api | API Key | `x-api-key` header — custom validation service |
| waves-cft | API Key | AWS API Gateway key |
| waves-tax TAXONOFY | API Key | `x-api-key` header |

## Role Hierarchy (Waves PHP backends)

```
ROLE_SMP_ADMIN
  └─ ROLE_COMPANY_ADMIN
       └─ ROLE_{PRODUCT}_ADMIN    (e.g. ROLE_CSRD_ADMIN, ROLE_CFC_ADMIN)
            └─ ROLE_{PRODUCT}_USER
                 └─ ROLE_{PRODUCT}_READ_ONLY
```

## Key Cross-System Integrations

| From | To | Purpose |
|------|----|---------|
| `waves-csrd-backend` | `waves-smp-backend` | Fetch company address data (`/api/companies/{id}/address`) |
| `waves-tendering-backend` | SMP Calculation API | Distance + emission calculations (`/api/v1/calculate/*`) |
| `waves-tendering-api` (Lambda) | `waves-cft` (Lambda) | Emission calculation per order leg |
| `waves-tendering-api` (Lambda) | Routing service | Distance estimation per leg |
| `waves-cft` (Lambda) | Distance API (openrouteservice) | Geolocation & routing |
| PCI `carbon-footprint-service` | EFMS (Emission Factor Matching Service) | Component emission factors |
| PCI `carbon-footprint-service` | EcoTransit SOAP API | Transport emission factors |
